SQL Operations with Pandas

Published: Sat 13 August 2016
By Andrew Burnie

In pandas.

tags: python pandas joins SQL

The below provides a guide showing how common SQL operations can be performed using pandas

Creating the Data

First we import NumPy and pandas:

Inner Joins

To perform a inner join we use the function pd.merge(DataFrame1, DataFrame2).

To be explicit, we add the argument on = to specify which column will be used to join the tables:

In [5]:
pd.merge(frame1, frame2, on = 'id')
Out[5]:
id price color
0 ball 12.33 red
1 pencil 11.44 white
2 pencil 11.44 red
3 pen 33.21 black

Sometimes the column names will differ across tables. We use left_on for left argument DataFrame1's column and right_on for DataFrame2's column:

In [6]:
pd.merge(frame1, frame2a, left_on = 'id', right_on = 'identification')
Out[6]:
id price color identification
0 ball 12.33 red ball
1 pencil 11.44 white pencil
2 pencil 11.44 red pencil
3 pen 33.21 black pen

To perform an inner join using the row index, we use the function:

pd.merge(DataFrame1, DataFrame2, left_index = True, right_index = True):

In [7]:
pd.merge(frame1, frame2, left_index = True, right_index = True )
Out[7]:
id_x price color id_y
0 ball 12.33 white pencil
1 pencil 11.44 red pencil
2 pen 33.21 red ball
3 mug 13.23 black pen
4 ashtray 33.62 orange desk

We can instead use DataFrame1.join(DataFrame2) if both of the columns being used for joining are row indexes. Although this is simpler syntax, it requires that there are no identically labelled columns.

Other Join Types

We use the how = argument when we want other join types.

For inner joins, we could have included how = 'inner' to make our code more explicit. This was not necessary, as inner joins are performed by default.

For a left join, we specify how = 'left':

In [8]:
pd.merge(frame1, frame2, on = 'id', how = 'left')
Out[8]:
id price color
0 ball 12.33 red
1 pencil 11.44 white
2 pencil 11.44 red
3 pen 33.21 black
4 mug 13.23 NaN
5 ashtray 33.62 NaN

Note the insertion of NaN where values are missing

For a right join, we specify how = 'right':

In [9]:
pd.merge(frame1, frame2, on = 'id', how = 'right')
Out[9]:
id price color
0 ball 12.33 red
1 pencil 11.44 white
2 pencil 11.44 red
3 pen 33.21 black
4 desk NaN orange

For a outer join, we specify how = 'outer':

In [10]:
pd.merge(frame1, frame2, on = 'id', how = 'outer')
Out[10]:
id price color
0 ball 12.33 red
1 pencil 11.44 white
2 pencil 11.44 red
3 pen 33.21 black
4 mug 13.23 NaN
5 ashtray 33.62 NaN
6 desk NaN orange

Replacing Values

To replace values, we could use .apply() and then assign the resulting value to the orignal column.

An easier alternative would be to use .replace(), which accepts a dictionary as input. The dictionary serves as a mapping, where the keys are the original values and the values are the new values.

In [15]:
#Using .apply(), we would set dup_frame['color'] =

dup_frame['color'].apply(lambda x: 'a' if x == 'white' else 'b')

#Alternatively, we could set dup_frame =

dup_frame.replace({'white': 'a', 'red': 'b'})
Out[15]:
color value
0 a 2
1 a 1
2 b 3
3 b 3
4 a 2

We can also use .replace() to replace NaN with some default value:

Mapping

Mapping can be used to change (using .rename()) the names in the column index (columns =) or the row index (index =):

In [17]:
frame3 = frame1.copy() #Need .copy(), otherwise changing frame3 changes frame1

row_index = {0: 'one', 1: 'two', 2: 'three', 3: 'four', 4: 'five'}

column_index = {'id': 'product', 'price': 'GBP Price', 'category': 'group'}

frame3.rename(index = row_index, columns = column_index, inplace = False)
Out[17]:
product GBP Price
one ball 12.33
two pencil 11.44
three pen 33.21
four mug 13.23
five ashtray 33.62

Note that adding inplace = True would mean that the original DataFrame frame3 is changed.

The function map() can also be used to add a new column of values, using their relationship with the values of another column:

In [18]:
frame3['category'] = frame3['id'].map({'ball':'a', 'pencil': 'b', 'pen': 'c', 'mug': 'd',
                                       'ashtray': 'd'})

frame3

1 pencil 11.44 b
2 pen 33.21 c
3 mug 13.23 d
4 ashtray 33.62 d

We can define our own aggregator function to apply, using function agg().

If we use a list as input, then a list of the different aggregate values is returned:

In [22]:
def min_max(series):
    return str(series.min()) + ' to ' + str(series.max())

frame4.groupby('category').agg(['mean', min_max])
Out[22]:

s 22.325 11.44 to 33.21

String Manipulation

The following string manipulations are not specific to pandas, but can be used more generally in Python. The below is useful for splitting or combining columns of string data in a table:

Suppose that we have the following sample text:

In [23]:
sample_text = '10 Downing Street, SW1A 2AA, London'

We can split this text up to form a list, using ',' as a separator.

To remove leading and end spaces, we can use .strip():

In [24]:
address_list = [s.strip() for s in sample_text.split(',')]

address_list
Out[24]:
['10 Downing Street', 'SW1A 2AA', 'London']

Similarly we can join text using +

In [25]:
address = '10 Downing Street'

postal_code = 'SW1A 2AA'

address + ', ' + postal_code
Out[25]:
'10 Downing Street, SW1A 2AA'

We can use .join() to join an entire list using a given separator - here | :

In [26]:
'|'.join(address_list)
Out[26]:
'10 Downing Street|SW1A 2AA|London'

More on Python list functions can be found here

Regular Expressions

We can also perform string search and manipulation using regex if we import the in-built Python module re:

In [27]:
import re

  • re.split(regex, text): splits up text and puts results in a list, using the given regex as a separator

  • re.findall(regex,text): gives list of substrings in text meeting the requirements of the regex
